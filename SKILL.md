# AutoShop Market — редизайн и UI-паттерны

## Контекст проекта

Стек: React + Vite + TypeScript, Tailwind CSS, Framer Motion, Supabase backend, деплой на Vercel.
Сайт: autoshop-market.vercel.app. Аудитория: Украина, покупатели автозапчастей и тюнинг-аксессуаров.
~65 000 товаров, 24 категории, десятки подкатегорий.

Артём получает от Claude Code готовые файлы целиком, а не патчи/инструкции — всегда выдавай полный код компонента/файла, не фрагменты "вставь сюда".

## Фирменный стиль (не менять без явного запроса)

- **Логотип**: "AUTOSHOP" — "AUTO" тёмным/белым, "SHOP" оранжевым акцентом
- **Основной цвет**: фиолетовый градиент (~#7c3aed → #a855f7), используется в hero-секции, кнопках "Купити", шапке
- **Акцентный цвет**: оранжевый (~#f97316) — для скидок, бейджей, вторичных CTA, части логотипа
- **Фон карточек**: белый/светло-серый, товар на нейтральном фоне
- **Кнопки**: скругления ~8-12px, залитые фиолетовым для основного действия, контурные для вторичного
- **Шрифт**: жирный sans-serif для заголовков, обычный для текста
- Язык интерфейса: украинский

## Проблема, которую решает этот скилл

Плоский список категорий (24 штуки) и десятки подкатегорий выведены прямо на главную страницу сплошной стеной кнопок-чипсов. Это создаёт когнитивную перегрузку: пользователь видит 30-40+ кликабельных элементов до того, как доходит до товаров. Baymard Institute и конверсионные исследования 2026 года показывают: плоский список из 20+ пунктов воспринимается тяжелее, чем 6-7 пунктов верхнего уровня, каждый из которых разворачивается в дропдаун/mega-menu. Правило рабочей памяти — 7±2 элемента на один уровень восприятия.

## Целевая структура навигации

### 1. Шапка (Header) — sticky
- Остаётся видна при скролле (`position: sticky; top: 0`)
- Логотип слева, поиск по центру, кнопка "Кошик" справа (уже есть)
- Добавить пункт **"Каталог"** с mega-menu по клику/наведению — именно сюда переезжает полный список категорий и подкатегорий, а не на главную страницу
- На мобильном — гамбургер-меню вместо полного списка в шапке

### 2. Mega-menu (выпадающая панель категорий)
- Открывается по клику на "Каталог" в шапке (не занимает место на странице, пока не вызвана)
- Двухколоночная (или трёхколоночная) структура: слева — список из 24 категорий верхнего уровня, справа — подкатегории категории, на которую наведён/выбран курсор
- Опционально: иконка на каждую категорию верхнего уровня для более быстрого сканирования
- На мобильном — drill-down на весь экран: тап по категории показывает список подкатегорий с кнопкой "назад", а не аккордеон с раскрытием всех сразу

### 3. Главная страница — освобождается от стены чипсов
Порядок блоков сверху вниз:
1. Промо-полоса (уже есть: бесплатная доставка / скидки)
2. Hero: оффер + текст + кнопка "Обрати категорію" + телефон + счётчики (уже есть, оставить)
3. 4 иконки-преимущества (доставка/гарантия/поддержка/скидки — уже есть)
4. **"Обери напрямок"** — вместо плоского списка ссылок: 6-8 крупных визуальных плиток с фото по самым популярным направлениям (автохимія, диски, тюнінг-оптика и т.д.), с кнопкой "Всі категорії →", которая открывает mega-menu. Плитка = фото + название, без длинного текста
5. Далее — популярные/новые товары, баннеры акций и т.д.

Никогда не выводить полный список из 24 категорий + все подкатегории на главную странице сплошным полотном — это единственное жёсткое правило этого скилла.

### 4. Хлебные крошки (Breadcrumbs)
Обязательны на страницах категории и товара: `Головна → Автомобільні диски → Ковані диски R17 → Kabeast Forged V6`. Помогают не терять контекст и улучшают SEO/индексацию.

### 5. Страница категории/каталога
- Слева сайдбар: активные подкатегории (уже частично реализовано как чипсы под фильтрами — оставить, это ок на уровне одной категории, не всего каталога) + фильтр по цене (слайдер)
- Сетка карточек 4-6 в ряд (сейчас 6 — ок для десктопа, на мобильном 2)
- Карточка товара в сетке: фото на нейтральном фоне → бейдж "Sale"/скидка при наличии → название → цена (зачёркнутая старая + новая при скидке) → рейтинг звёздами с числом отзывов → кнопка "Купити" на всю ширину

### 6. Страница товара
- Слева: галерея фото (главное фото + миниатюры)
- Справа: категория-ссылка, название, рейтинг, цена, выбор количества, кнопки "Додати в кошик" (контурная/вторичная) и "Купити зараз" (залитая, основной CTA) — сейчас есть только одна кнопка "Додати", добавить вторую для прямой покупки
- Ниже: аккордеон/вкладки — Характеристики, Опис, Доставка, Оплата, Умови (частично уже есть)
- Блок відгуків: средний рейтинг + разбивка по звёздам столбиками (5→1) + список отзывов с именем, датой, текстом, "Це було корисно?"
- **Мобильная версия**: sticky-бар внизу экрана с ценой и кнопкой "Купити", чтобы не нужно было скроллить наверх для покупки

## Чего избегать
- Не выводить длинные плоские списки ссылок вне mega-menu
- Не перегружать mega-menu лишними баннерами/декором — только категории, подкатегории, по желанию иконки
- Не менять фирменные цвета (фиолетовый/оранжевый) без явного запроса
- Не полагаться на память при вёрстке — если нужны актуальные факты о библиотеках/API, использовать инструменты поиска
