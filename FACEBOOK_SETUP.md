# Настройка автопостинга в Facebook

Система уже готова: очередь постов лежит в `marketing/fb-posts.json`, картинки — в `public/fb/`, публикацией занимается GitHub Action «Facebook: публікація поста» (по будням в 11:00 по Киеву, один пост за запуск).

Чтобы всё заработало, нужно один раз получить токен страницы и добавить два секрета в GitHub. Делается за ~15 минут.

## Шаг 1. Создать приложение Meta

1. Зайди на **developers.facebook.com** под своим аккаунтом Facebook (тем же, с которого создана страница).
2. Если попросят зарегистрироваться как разработчик — соглашайся, это бесплатно.
3. Нажми **My Apps** (справа вверху) → **Create App**.
4. Тип использования: выбери **Other** → далее тип приложения **Business** → **Next**.
5. Название приложения — любое, например `autoshop-poster`. Email оставь свой. **Create App**.

## Шаг 2. Получить токен страницы

1. Открой **Graph API Explorer**: developers.facebook.com/tools/explorer
2. Справа в поле **Meta App** выбери своё приложение (`autoshop-poster`).
3. В поле **User or Page** нажми → **Get User Access Token**.
4. В списке разрешений (Permissions) добавь три:
   - `pages_show_list`
   - `pages_read_engagement`
   - `pages_manage_posts`
5. Нажми **Generate Access Token** — Facebook попросит подтвердить вход и выбрать страницу. Выбери свою страницу и разреши всё, что предлагает.
6. Теперь продлим токен (иначе он умрёт через час):
   - Рядом с токеном нажми на синюю иконку ⓘ → **Open in Access Token Tool**.
   - Внизу нажми **Extend Access Token** — получишь токен на 60 дней. Скопируй его.
7. Возвращайся в Graph API Explorer, вставь продлённый токен в поле **Access Token**.
8. В строке запроса (где написано `me?fields=id,name`) введи: `me/accounts` и нажми **Submit**.
9. В ответе найди свою страницу. Там будут два значения:
   - `"access_token": "EAAG..."` — это **токен страницы** (он бессрочный, потому что получен из продлённого токена)
   - `"id": "1234567890"` — это **ID страницы**
   Скопируй оба.

## Шаг 3. Добавить секреты в GitHub

1. Открой репозиторий на GitHub → **Settings** → **Secrets and variables** → **Actions**.
2. **New repository secret**:
   - Name: `FB_PAGE_TOKEN`, Secret: токен страницы из шага 2.9
3. Ещё раз **New repository secret**:
   - Name: `FB_PAGE_ID`, Secret: ID страницы из шага 2.9

## Шаг 4. Проверка

1. GitHub → **Actions** → слева «Facebook: публікація поста» → **Run workflow** → зелёная кнопка **Run workflow**.
2. Через минуту-две проверь страницу Facebook — там должен появиться первый пост из очереди.
3. Если workflow красный — открой его лог и пришли текст ошибки Claude.

## Как это живёт дальше

- Каждый будний день в 11:00 публикуется **один** пост из очереди (первый со статусом `approved`).
- Когда очередь пустеет — ничего страшного не происходит, workflow просто молчит. Раз в неделю говоришь Claude «подготовь посты на неделю» — он пополняет очередь (тексты + картинки) и пушит.
- Пост не понравился до публикации? Скажи Claude — он поправит текст или поменяет статус на `draft`.
